# 人才政策智能问答系统

## 一、整体定位

为政府人才服务门户打造一个"惠才通"智能问答助手：用户可通过顶部导航或右下角悬浮按钮唤起问答弹窗，弹窗内既能浏览常见问题（FAQ），也能与 AI 自由对话；支持一键全屏，方便阅读长答复和材料清单。

视觉采用**政务严肃风**：以中国红 + 政务蓝为主色，配白色卡面与金色点缀，体现权威感与服务温度。

---

## 二、页面结构

### 1. 主页（Index）

- **顶部导航栏**：政府门户标识 + 主导航（首页 / 人才政策 / 申报指南 / 政策问答 / 联系我们）；"政策问答"项高亮，点击直接打开弹窗。
- **首屏 Hero**：政务红蓝渐变背景 + 主标题"汇聚天下英才，共建美好家园" + 副标题 + 双 CTA（"立即咨询人才政策" / "查看申报指南"）。
- **政策概览卡片区**：四张卡片对应四大主题（人才引进与落户 / 人才补贴与奖励 / 住房保障 / 子女教育与配偶就业），每张卡片配图标、简介、"问问 AI"按钮，点击后打开弹窗并自动定位到对应主题。
- **数据成效区**：累计引进人才数、发放补贴金额、服务企业数等数字展示，强化公信力。
- **页脚**：政府联系方式、办事大厅地址、备案信息、友情链接。

### 2. 悬浮问答入口

- **右下角悬浮按钮**：圆形红底白字"答"+ 文字气泡"政策有问题？点我"，常驻所有页面；首次进入有轻微脉冲动效引导注意。

## 三、问答弹窗（核心模块）

### 弹窗形态

- **默认形态**：居中模态窗，宽 ~860px、高 ~640px，圆角 + 阴影，背景虚化遮罩。
- **全屏形态**：点击右上角"全屏"图标后，弹窗扩展至整个视口，便于查阅长内容；再次点击恢复。
- **关闭**：右上角 ✕ 或 ESC。

### 弹窗布局（左右分栏）

```text
┌─────────────────────────────────────────────────────────────┐
│  🛡 惠才通·人才政策智能问答         [全屏]  [✕]              │
├──────────────────┬──────────────────────────────────────────┤
│ 主题分类         │  对话区                                  │
│ ▸ 全部           │  ┌────────────────────────────────────┐ │
│ ▸ 人才引进/落户  │  │ AI: 您好！我是惠才通助手……        │ │
│ ▸ 人才补贴       │  └────────────────────────────────────┘ │
│ ▸ 住房保障       │                                          │
│ ▸ 子女教育/配偶  │  常见问题（点击直接提问）               │
│                  │  • A 类人才落户需要哪些材料？           │
│ 热门搜索词       │  • 博士毕业生安家费多少？               │
│ #安家费 #落户    │  • 人才公寓怎么申请？                   │
│ #人才公寓        │  • 配偶能随调到事业单位吗？             │
│                  │                                          │
│                  │  ┌─────────────────────────┐ [发送]     │
│                  │  │ 请输入您的问题…         │            │
│                  │  └─────────────────────────┘            │
│                  │  💡 回答仅供参考，正式以官方文件为准    │
└──────────────────┴──────────────────────────────────────────┘
```

- **左栏（侧边导航）**：主题分类（点击切换 FAQ 列表与 AI 上下文）、热门搜索词标签。
- **右栏上方（对话区）**：流式渲染 AI 回答，支持 markdown（标题、列表、链接），用户气泡靠右、AI 气泡靠左，AI 头像为政府徽标样式。
- **右栏中部（FAQ 快捷入口）**：根据当前主题动态展示 6–8 条常见问题，点击即作为用户提问发送。
- **右栏下方（输入框）**：多行文本输入 + 发送按钮 + Enter 发送 / Shift+Enter 换行；下方提示语免责声明。
- **顶部工具栏**：标题徽标、全屏/退出全屏按钮、清空对话按钮、关闭按钮。

### 交互细节

- **首次打开**：AI 自动发送欢迎语 + 引导点击 FAQ 或自由提问。
- **流式回答**：token-by-token 渲染，回答中显示"AI 正在思考…"动画。
- **多轮上下文**：保留当前会话历史一并传给 AI，支持追问。
- **复制回答**：每条 AI 消息底部"复制"按钮。
- **错误提示**：429（限流）/402（额度）友好 toast 提示。
- **全屏模式**：左栏可显示更多分类与热词，对话区文字加大、行距加宽。
- **响应式**：移动端弹窗自动全屏，左栏折叠为顶部下拉。

## 四、常见问题示例（预置内容）

每个主题各预置 6–8 条，覆盖最关心的场景，例如：

- **人才引进与落户**：A/B/C 类人才认定标准？落户办理流程？所需材料清单？办理时长？
- **人才补贴与奖励**：博士/硕士安家费金额？生活补贴发放周期？创业资助上限？如何申报？
- **住房保障**：人才公寓申请条件？租金标准？购房补贴比例？申请入口在哪里？
- **子女教育与配偶就业**：子女入学优待政策？跨区入学如何办理？配偶随调流程？

实际答案由 AI 基于系统提示词中的政策知识库生成。

## 五、技术实现要点（开发参考）

- 启用 **Lovable Cloud + Lovable AI Gateway**，默认模型 `google/gemini-3-flash-preview`。
- 新建 Edge Function `chat`：注入"人才政策助手"系统提示词（含四大主题政策要点框架 + 严谨免责说明），SSE 流式转发。
- 前端 `PolicyChatDialog` 组件：基于 shadcn `Dialog`，自定义全屏切换状态；使用 `react-markdown` 渲染 AI 回答；通过 fetch + ReadableStream 解析 SSE。
- 设计 token：在 `index.css` 定义 `--gov-red`、`--gov-blue`、`--gov-gold` 等 HSL 变量，并在 `tailwind.config.ts` 暴露；所有组件走语义化 token，无硬编码颜色。
- 路由保持单页（`/`），弹窗通过全局状态控制（简单 React Context 即可）。
- FAQ 数据以 `src/data/faq.ts` 形式静态维护，方便后续替换为接口。
